Posture technique

TAXEM est une initiative de technologie civique en phase de développement. Il est conçu dans une optique de préparation à la certification (certification-aware), ce qui signifie que son architecture sous-jacente intègre des points de raccordement propres pour la transmission, mais il n'est **pas encore certifié** par l'Agence du revenu du Canada (ARC) ni par Revenu Québec (RQ).

Pour maintenir une séparation stricte des responsabilités, le cœur du moteur de calcul ne contient aucun code de transmission, aucune connexion directe aux réseaux gouvernementaux, ni aucun stockage de clés d'identification de transmission. La certification formelle sera poursuivie uniquement par les canaux officiels dans une version ultérieure.

Mesures de validation

Nous croyons fermement que les calculs fiscaux doivent être transparents et auditables. Pour garantir l'intégrité du moteur de calcul TAXEM, nous maintenons une suite de tests rigoureuse exécutée automatiquement contre les données historiques et les paramètres d'impôt gelés.

Garde-fous d'admissibilité de transmission

La version v1 de TAXEM intègre un module de validation d'admissibilité. Ce module intercepte les résultats de calcul avant toute tentative de transmission et vérifie que la déclaration du contribuable répond aux critères requis.

Pendant notre phase de développement actuelle, le module valide les règles suivantes :

  • Portée limitée à l'Ontario : Seules les déclarations des résidents de l'Ontario (avec les formulaires ON428 et ON479) sont traitées. Tous les autres profils provinciaux sont écartés de manière sécurisée.
  • Blocage des situations complexes : Les cas nécessitant des formulaires complexes (tels que la double juridiction ou des situations de faillite en cours d'année) sont bloqués.
  • Verdict par défaut : Comme le logiciel n'est pas encore certifié, le module renvoie un statut needs_review (vérification requise) plutôt que electronic_ok pour toutes les déclarations compatibles.
